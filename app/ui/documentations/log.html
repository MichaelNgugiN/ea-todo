Introduction

THE ERROR LOG TRACKER MANUAL

The Error Logger is an application that fetches both "application and system errors from critical servers" that need constant monitoring such as the cluster servers. 

Languages used:

⦁ The main interface runs on HTA for the front-end and VBScript for back-end.

⦁ The core of the system runs on PowerShell which is used to access the servers remotely and fetch their error logs saving them as csv files. 

⦁ The reporting is done on an Excel application using Visual Basic that reads from CSVs created by the PowerShell script above.

Step 1) Installing the application and launching it.

Access the folder where the application is running from. Currently:  \\10.80.101.25\Application\The EventLogger\

This can be done by opening Run by pressing Windows key + R and pasting the path then clicking on OK.

Step 2) creating a shortcut

Copy the application shortcut to any location on your machine, for instance, your desktop. 

Double click on the shortcut you have copied to your machine. Click on Run on the prompt that will appear.

Step 3) Allowing access

This will copy all necessary files (such as CSS files and scripts) from the server to your local machine to allow smooth running of the application.
